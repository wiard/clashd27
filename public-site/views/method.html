How CLASHD27 finds gaps

CLASHD27 is a collision engine that discovers untested connections between AI research subfields. It runs continuously, scanning new papers and producing scored, verifiable gap reports.

Pipeline

01
Sample
Papers are ingested daily from arXiv, Semantic Scholar, OpenAlex, and PubMed/EuropePMC. Each paper is classified by method (observational, computational, experimental), surprise level, and semantic domain.
02
Cube
A 3×3×3 cube topology organizes papers into 27 cells. A clock selects one cell every 5 minutes. Autonomous agents traverse the cube, forming bonds when they meet in the same cell.
03
Collide
When agents bond across layers, the engine generates golden collisions — pairs of papers from maximally distant cells. These are scored for collision strength, method distance, semantic distance, and surprise.
04
Screen & Verify
Collisions pass through red-flag screening, satisfaction scoring, structure validation, and multi-source verification (PubMed, clinical trials, NIH grants). Only gaps scoring above threshold survive.
05
Deep Dive
High-scoring gaps (score ≥ 75) trigger deep-dive investigations that cross-reference additional databases and compute credibility scores.
06
Publish
Surviving gaps are published with full evidence chains, proposed experiments, risk assessments, and scoring breakdowns. All state is append-only and reproducible.

Data Sources

  • arXiv — Preprints across CS, physics, math, biology
  • Semantic Scholar — Academic graph with citation context
  • OpenAlex — Open catalog of scholarly works
  • PubMed / EuropePMC — Biomedical literature
  • ClinicalTrials.gov — Trial registry for validation
  • NIH Reporter — Funded grant verification

Design Principles

Budget-capped: Every API call is metered. The engine enforces daily limits to stay reproducible and cost-efficient.

Append-only: State files are never overwritten destructively. Each gap, finding, and verification is recorded permanently.

Reproducible: Given the same input data and the same tick, the pipeline produces the same output.
